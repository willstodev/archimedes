# Legacy V1 Migration

Legacy JSON, CSV, Markdown, and Google Sheets data must be treated as import sources, not as operational state.

Migration order:

1. Inventory legacy files without mutating them.
2. Map fields to the PostgreSQL schema.
3. Import idempotently with dry-run support.
4. Preserve source identity and row provenance.
5. Reconcile counts, duplicates, and statuses.
6. Write a reconciliation report.
7. Switch normal reads and writes to PostgreSQL only after verification.

Never delete or overwrite V1 data during migration. Never mark an application or outreach message as sent based only on generated Markdown.
